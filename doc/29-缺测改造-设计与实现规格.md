## 1. 修复总口径（本特性三条硬约定）

1. **任一约定核心交易指标缺测 → 放弃买入；若该票已在持仓 → 一律卖出**（最保守，宁可错过不买盲）。
2. **缺测用显式标记字段表达**（如 `data_missing=true` + 缺测原因），**不靠裸 null**——执行侧现把「真值缺失」与「正常空值」都落 None、不可区分。
3. **隔夜不在今日 watchlist 的持仓不强卖**，走现有止损/炸板/盘口逻辑（不在名单 ≠ 缺测，避免无差别清仓）。
4. 受 A 股 **T+1** 物理约束：「持仓则卖」只能对**已过 T+1 的可卖单元**生效（当日买入票当天卖不掉）。

## 2. 约定核心交易指标集合（缺测①的范围）

任一缺测即触发「放弃买入/持仓卖出」（实现时以此为准、可再细调）：
`close`（信号日基准价）、`board_level`（连板高度）、`tradable_flag`、`leader_strength_score`、`continuation_prob`（续板档）、`seal_ratio_pct`/`float_mktcap`（封流比及分母）、`first_board_vol`/`volume_ratio`（量能比及分母）、`return_5d/10d_pct`（位置）、封板时序（`first/last_limit_time`/`open_times`）。
> 已知保守取舍：停牌导致 `return`/封板时序缺测的票也会被放弃/卖（负责人已确认接受）。

## 3. 已查明的关键事实（避免重复调研）

- **所有缺数据都能经 Tushare 取到**（行情/因子/炸板池全在），无「Tushare 根本拿不到」项。
- **S-10/1.2.0 取错源**：`limit_list_d`（Tushare 涨停统计，OPTIONAL_APIS 已每日抓取）含数值型 `limit_times`（连板高度）、`first_time`/`last_time`/`open_times`（封板时序），但代码只用「开盘啦 kpl_list 的 status 文本」；且 `open_times` **根本没写进 KPL 请求字段** → 恒 None。`limit_list_d` 数据已在 `optional_payload`，**未按 ts_code 合并进个股 compact 行**。
- **「持仓则卖出」当前完全无路径**：`prior_provider`（`app/run.py:206-251`）不搬 `tradable_flag`/缺测标记 → `SignalPrior` 里丢失；`sell_decider` 卖出全靠 OrderBook 实时盘口扳机，无缺测分支；`_evaluate_and_sell_unit`（`main.py:1120-1167`）在 `book is None` 时默认 HOLD（`:1145`）——与「缺测则卖」相反。

### A1 · 缺测 sentinel 契约（先做，后续都依赖它）
- **现状**：`app/schemas/limit_up_watchlist.py` 有 `tradable_flag: str = "TRADABLE"`（line 63 附近）、`action: str | None`；各指标 `X | None`，缺测落裸 null、与正常空值不可区分。

### A2 · S-3 炸板池接口 FAILED 保守化（闸门安全，优先）
- **现状**：炸板/跌停池 `_safe_query(required=False)`（`:1770-1777`）失败时 `return []`（`:1822-1826`），与「真实零炸板」同形；`broken_board_rate_pct` 被算成 `0.0`（`:2585-2595`）；`sentiment_gate.py:277` 据此放行（`broken=0.0` 不触发硬否决）→ 真实高炸板退潮日可放行开新仓。
- **改法**：参照已修的 **S-11**（区分「未知」与「真低」）：让 `_safe_query` 的 FAILED 可被上层感知（如返回带 data_quality 标记，或单独记录 FAILED 集合）；FAILED 时 `broken_board_rate_pct` 置 **None（未知）而非 0.0**；`sentiment_gate` 在炸板池/跌停池**未知**时按**市场级保守**处理（缺证据 → 倾向退潮/谨慎、禁开新仓），**不得把 FAILED 当零炸板放行**。
- **验收**：单测——炸板池 query 抛异常（FAILED）时，闸门不输出「健康/参与」、不放行开新仓；真实空炸板（EMPTY）仍正常判健康。

### A3 · S-2 信号日 T 行缺失不再用 T-1 冒充
- **现状**：`_technical_indicators` 兜底 `if not daily_rows`（`:1909-1914`）只在整段窗口为空时触发；本地有 T-1 但缺 T 时不触发兜底；`_calculate_indicator` 取 `latest = rows[-1]`（`:2061-2093`）即 T-1 行，`close/vol/pct_chg/return` 全回退 T-1，无 `latest.trade_date==T` 校验。
- **改法**：①兜底触发条件改为「窗口内**无信号日 T 行**即补取 Tushare `daily`」（不只整段空）；②`_calculate_indicator` 增 `latest.trade_date == 信号日T` 校验，补不到 T 行则把 `close/vol/pct_chg/return_5d/10d`（及衍生 `first_board_vol`）置**缺测**（不取 T-1）。
- **验收**：单测——本地仅有 T-1 行、Tushare 也取不到 T 行时，相关指标为缺测（非 T-1 值），不污染 watchlist。

### A4 · S-9 `_window_return` 按交易日跨度校验
- **现状**：`_window_return`（`:4701-4705`）按「非空 close 计数」取 `filtered[-window-1]`；窗口 `start_date = trade_date - timedelta(days=...)`（`:1898-1900`）按日历天 → 停牌缺口致 `return_5d/10d` 跨多于 N 个交易日、空间高度高估。
- **改法**：`_window_return` 增「非空 close 数足够 **且** 实际跨度 ≤ N 个交易日」校验，不足/超跨度即该因子置**缺测**；窗口按交易日加载（够 N+缓冲个交易日）。
- **验收**：单测——窗口内有停牌缺口致样本<N 或跨度>N 时，`return_5d/10d` 为缺测。

### A5 · S-10 连板高度切 `limit_list_d.limit_times` 主源

### A6 · 1.2.0 封板时序切 `limit_list_d` 权威源（修 `open_times` 恒空）
