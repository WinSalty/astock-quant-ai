## 二、ROUTE-1 门控口径决策（已拍板）

`QMT_AUCTION_TIMING_ENABLED=false` 时，「竞价强开追」战法**完全停用**，包括其在 9:25–9:30 定盘段以开盘后单身份的下单——
与运维「该开关关 = 该战法不实盘」的预期一致，go-live 初期最保守。其余战法（打板跟买、均线低吸）的开盘后下单不受此开关约束。

## 三、未修复：按性质上交决策（不擅自默认处理）

> 以下为「真机实测才能定 / 跨两仓契约 / 仅告警不阻断 / 需负责人定口径」的项，按项目硬约定**不做看起来有值的默认处理**。

### 3.1 必须真实下单 / 竞价窗才能观测（开通道后首批真实委托一并核对）

- **A4 竞价段封单口径**：集合竞价段 `bidVol[0]` 是虚拟匹配量、真实涨停封单在买侧残余 `bidVol[1]`，虚拟封单/封流比在竞价段被严重低估（约 300 倍）。买入侧（虚拟封单因子）与卖出侧（封板续持/减仓判定）同源，启用竞价择时前须先定竞价段封单取数口径。
- **A4 延伸**：实测竞价段成交量为 0，竞价量能比与分时重心两个因子在竞价段结构性失效——启用竞价择时前须确认竞价段用什么字段判强弱。
- **A8**：`order_stock` 同步失败返回值（代码按 `<0` 判失败；若实际返回 0 会被当有效委托号推进，须真机制造废单观测）。
- **A9①**：`cancel_order_stock` 对已成/不存在单的返回与是否抛异常（代码未检查 cancel 返回码，仅异常兜底）。
- **A9②**：状态码 255 是否可能为「活单」（代码映射为已废终态，若实为活单则卖单复位重挂会出现同票两笔活卖单，靠券商可用量/超量拒单兜底）。
- **A10**：券商侧 `order_remark` 实际截断长度（影响信号日回填的留痕可读性，有日历反推兜底）。
- **下单/持仓/成交量纲=股**：代码全程按股处理、按 100 股取整；首批真实委托须确认 `order_volume`/`traded_volume`/持仓 `volume` 量纲与代码假设一致（`get_full_tick` 行情量=手已实测确认，交易域=股为 QMT 惯例但未真机下单确认）。

### 3.2 跨两仓契约 / 仅告警 / 复盘统计口径（交决策）

- **两系-9**：回流同步未区分「永久失败(4xx)」与「瞬时失败(5xx/网络)」、无死信终态，永久失败行每次重投都失败。需定死信口径。
- **两系-4**：盘前名单只在响应顶层目标交易日存在时做 fail-closed，顶层缺失时无单条 trade_date 第二道校验。需确认信号侧契约是否保证顶层恒在。
- **执行-9**：弱封减仓以涨停价挂半仓（默认阈值关、不触发）；启用前须定封板期减仓挂单价口径（避免砸自家封板），依赖 A4。
- **对账告警类（执行-13/14/15/20）**：出入金未从成交净额抵消、高换手相对容差、UNKNOWN 方向成交早退、碎单合成键折叠——均**仅告警不阻断交易**，属容差/告警口径，待统一定。
- **REC-1（成交明细去重）**：缺成交编号又缺成交时间且同量同价的碎单会被合成同键折叠少计一笔——属成交明细/复盘数据完整性（持仓/资金以券商权威重建、不受影响），且依赖「QMT 是否真会同时缺成交编号与时间」（真机倾向于成交编号有值），与 A8/A9 一并真机核对。

## 四、启用「竞价择时」前的前置（依赖顺序）

1. 先定 **A4 竞价段封单与量/重心取数口径**（§3.1），并真机竞价窗验证；
2. 同步核对：竞价择时开启后，定盘段（9:25–9:30）该战法的分时重心在「定盘段无新撮合、成交量大概率不变」下可能恒判不出上升趋势而持续不买——需与 A4 口径一并确认定盘段强弱判据；
3. `QMT_AUCTION_TIMING_ENABLED` 仅在以上口径确认（标记 verified）后才允许开启（启动期已有 fail-closed 强校验）。
