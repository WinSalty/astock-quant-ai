## 三、「绝不买入 ST」三层防御设计

ST 判定统一口径 `universe_filter.is_st_stock(is_st_flag, name)` = `(is_st_flag is True) or is_st_name(name)`
（显式 `is_st=True` 或证券名含 ST/*ST/退 即判 ST；最严格取向，杜绝任一信号漏判）。

- **可靠透传（堵 F08）**：信号侧 `is_st` 布尔 → `watchlist_item_to_selected` → SQLite `watchlist.is_st` 列(+迁移) →
  `mappers` 双向编解码 → `SelectedStockRow.is_st` → `TradableEntry.is_st/name` → `PlanRow.is_st/name` → `EntryDecision.is_st`。
- **第 1 层（盘前 universe）**：`watchlist_loader._process_row` 用统一口径把 ST 票剔出可交易名单（转观察）。
- **第 2 层（建仓路由）**：`entry_router._should_skip` 对 ST 票一律 SKIP（不产 BUY 决策、留痕）。
- **第 3 层（唯一下单点）**：`order_executor.place` 对 `decision.is_st` 真者**拒单**——所有买入（含转次优）必过此处，是最终硬保证。

> 任一层独立即可阻断，三层叠加确保即使 `is_st` 缺失/某层被绕过/未来改动，ST 标的也绝不会有买单发出。

**两轮 review 发现并修复的「修复未彻底/新引入」缺陷**（对抗式复核抓出）：
1. **[review·high] F02 同日回扣死路**：`on_stock_order` 先 `sync_status` 把部成撤单 fill-aware 收口为 `PART_TRADED`，原 `_maybe_revert_sell_unit` 按 `entry.state∈终态集合` 判定 → 部成撤单永进不来 release 分支、当日仍漏卖。**修**：改按【回报原始状态 `terminal_status`】(`rec.order_status`/`OrderStatus.ERROR`) 判定，零成交/部成统一按 `unfilled=plan-filled` 精确回扣。
2. **[review·medium] 多单在途零成交误清**：filled==0 走 `revert_selling` 整体清零 on_road，会误清同单元另一在途单冻结量。**修**：统一走精确回扣（`release_on_road_by_code`），仅 on_road 归零才复位 `SELLING→HOLDING`。
3. **[review·high·二轮] on_road 回扣非幂等**：同一卖单终态回调重复触发（拒单经 `on_order_error`+`on_stock_order` 双面回报 / 同帧重投）会重复回扣、误清兄弟在途单。**修**：`PositionUnit.released_sell_order_ids` 按 `order_id` 去重（每单只回扣一次），`refresh_state` 跨日重置去重集。
4. **[review·low] F19 全非正分退化等权**：top-N 真实分全为非正时 total<=0 落入「退化等权」把被钳 0 份额的弱/坏票又买回。**修**：区分「top 全缺强度(None)→合法等权」vs「top 有真实分但全非正→不分配(weights 空)」。
5. **[review·medium] `QMT_WRITE_QUEUE_STUCK_SECONDS=0` 被 `or 30.0` 吞**：显式关看门狗逃生口不可达。**修**：改 `is-not-None` 守卫（与 `write_queue_max`/`seal_ratio_min` 同口径）。

二轮 review 复核：3 个修复正确，1 个（on_road 回扣）补幂等后正确；其余无新缺陷。
