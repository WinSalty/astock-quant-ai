## 0. 修复统一约定（贯穿全程）

1. **口径①·数据偶尔不全（Tushare 能取到但偶发缺）**：约定一个**特殊值/null sentinel**；该票相关核心指标缺测即**放弃买入**；若该票**已在持仓则一律按卖出处理**（受 A 股 T+1 物理约束，仅对已过 T+1 的可卖单元生效）。
2. **口径②·Tushare 根本拿不到**：先研究其它取数渠道，再与负责人商量，**不擅自默认处理**。
3. **口径③·契约/跨两仓类**：给最合理方案+推荐，**交负责人决策，绝不默认处理或跳过**。
4. **口径④·回测类**：移交 [doc 28](28-回测方向问题记录.md)；LLM 选股下回测样本有限，代码修复优先级低、可暂不处理。

## 2. 修复顺序（分层 + 依赖 + 验收）

> 总依赖：**第 0 层（P0）先行** → 第 4 层契约决策中"sentinel 载体/版本"先于第 3 层数据缺测改造落地 → 其余各层内"先单仓、后跨仓"。每条均走 `AGENTS.md` 开发流程（规划→开发→测试→review→…）。

### 第 3 层 · 数据缺测统一改造（口径①：约定 null→放弃买入/持仓卖出）

> **一组系统性改造**，须信号侧产 sentinel + 执行侧三处接线一起做；**前置依赖第 4 层中"sentinel 载体/契约版本"决策**。

**A. 信号侧产出端（修取数 + 缺测置 sentinel）**
- `S-2`：修触发条件 bug——本地缺信号日 T 的 daily 行时**实时补取 T 日 daily**（`_query_indicator_api("daily")` 兜底改为"缺 T 行即触发"，并校验 `latest.trade_date==T`）；补不到才置缺测。位置 `limit_up_push_service.py:1909-1914,2061-2093`。
- `S-9` / `return_5d10d`：`_window_return` 增"非空 close 数足够 + 实际跨度≤N 个交易日"校验，不足即该因子置缺测；窗口按交易日而非日历天加载。位置 `:4701-4705,1898-1900`。
- `first_board_vol`/`float_mktcap`/`volume_ratio`：随上游缺测置 null（数据本身 Tushare 可靠，仅随 T 行缺口偶发不全）。
- `S-3`：炸板池/跌停池 `_safe_query` FAILED 与"真实空 EMPTY"区分，**FAILED 置"未知"而非 0**；闸门据"未知"按市场级保守（空仓/放弃开新仓），不把 FAILED 当真实零炸板放行。位置 `limit_up_push_service.py:1822-1826,2585-2595` + `sentiment_gate.py:277-279`。
- `S-10`：连板高度改/兜底取 `limit_list_d.limit_times`（可靠数值），替代/校验 kpl status 文本正则（**主源切换待决策，见第 4 层**）。
- 1.2.0 封板时序：修 `open_times` 恒 None（不在 `KPL_FIELDS`）；权威源对齐（kpl_list vs limit_list_d，**待决策**）。

**B. 约定 sentinel 与裁决载体（契约，待决策细项见第 4 层）**
- 复用 `tradable_flag`（现 `str` 默认 `TRADABLE`，新增枚举如 `DATA_MISSING`/沿用 `BLOCKED`）+ 新增显式"缺测原因"标记字段，避免执行侧逐字段 null 反推；升 `WATCHLIST_SCHEMA_VERSION`（现 1.2.0）。位置 `schemas/limit_up_watchlist.py:26,59,63`。
- `S-4/S-5`（FOCUS 先验非法/降级静默落 NULL）并入：在 `_do_persist_selected_stocks` 落表处（`limit_up_push_service.py:765` `blocked` 同处）追加"约定先验/指标缺测→`tradable_flag` 缺测 + `action=放弃观察`"分支并留痕。

**C. 执行侧放弃买入接线**
- 主接线点 `buy_prefilter`：`CandidateView` 增缺测字段、`_RULES` 追加 `_rule_data_missing`（命中 sentinel → `allowed=False`），获 loader/entry_router/order_executor 三层冗余。位置 `common/buy_prefilter.py:41-52,121`。
- `base.prior_gate_reason` 现"缺数据不误杀"(fail-open)需对**约定缺测 sentinel** 反转为 fail-closed（区分"普通降级 null"与"约定缺测"）。位置 `entry/strategies/base.py:170-190`。
- 注意：`remote_watchlist` 现**不映射 `action` 字段**，故约定走 `tradable_flag` 比 `action` 稳；裸 null 在 `remote_watchlist` 与真值不可区分，故用特殊 sentinel 比裸 null 可靠。位置 `watchlist/remote_watchlist.py:112-114`。

**D. 执行侧持仓卖出接线（当前完全无此路径）**
- 扩展 `SignalPrior`（`contracts/models.py`）增 `data_missing`/`tradable` 标记，并在 `prior_provider` 映射（`app/run.py:236-246`）从 `SelectedStockRow` 搬过来（须 `remote_watchlist` 先透传缺测标记）。
- `sell_decider` 前置短路区加"`prior.data_missing → CLEAR`"硬分支，**先于"盘口缺失→HOLD"早退**（`sell_decider.py:115-138`、`app/main.py:1145`），使缺测票即便无盘口也强制清。
- **守 T+1 优先**：当日买入票当天物理不可卖，缺测清仓只对已过 T+1 的可卖单元生效（`position_manager.py:348-365`）。

- **验收**：信号侧某票 T 行缺测 → 导出 `tradable_flag=缺测`；执行侧该票被 `buy_prefilter` 拒买；该票若为已过 T+1 的持仓 → 触发 CLEAR；当日买入票不被误卖。
