## 0. 一句话定位

**信号侧负责「看得准」（产出今天关注哪些票），执行侧负责「买得到、守得住、可复盘」。** 两侧物理隔离、HTTP 解耦。执行侧消费信号侧的 watchlist 只读契约 + 自采实时行情，**自主决定打不打 / 几点打 / 什么价 / 多少仓 / 何时卖**，独立下单并承担全部盈亏。

贯穿全局的硬口径（设计红线）：

1. **安全默认（fail-closed）**：任何契约残缺 / 行情断流 / 通道中断 / 数据不确定 → **只守仓、不开新仓**，宁可错过不可错做。
2. **守 T+1 双保险**：当日买入当日绝不可卖（业务闸 + 量闸两道）。
3. **三层幂等**：业务单号 + 台账唯一键 + 成交编号 traded_id，杜绝重复下单 / 重复入账。
4. **三层禁买闸**：绝不买入 ST、禁买四板及以上——盘前装载、建仓路由、唯一下单点三处叠加。
5. **时间口径统一**：全程 UTC naive，东八区换算只走 `common/time_utils.py`，**禁手工 ±8h**。
6. **一键熔断 + 高风险开关默认关**：`KILL_SWITCH` 只采集不下单；竞价择时、盘中卖出链默认关闭，实测/补数后才灰度开。

## 1. 一日交易时间线（端到端编排）

调度由 `app/scheduler.py` 的 `DailyScheduler` 按东八区钟点驱动，编排逻辑在 `app/main.py` 的 `Engine`。各动作幂等（`_fired` 记录当日已触发集合，`scheduler.py:95`）。

| 时刻（东八区） | 动作 | 入口 | 主要工作 |
|---|---|---|---|
| ≥ 08:55 | `PREWARM`（一次） | `Engine.prewarm` (`main.py:359`) | 连接守护建连 → 装载当日 watchlist → 推进持仓跨 T+1 → 用 QMT 权威重建持仓 → 抓**日初权益基线**（回撤熔断基准）→ 按强度归一**资金权重** |
| 09:15–09:30 | `RUN_AUCTION`（一次） | `Engine.run_auction` (`main.py:951`) | 启动 `AuctionPoller` 竞价轮询（阻塞至 9:30），逐帧算四因子 → 建仓路由；（可选）竞价段卖出评估 |
| 09:30–15:05 | `INTRADAY`（周期） | `dispatch(INTRADAY)` (`scheduler.py:219`) | 存储健康体检 → 下单通道心跳探活 → `sweep_ttl` 超时撤单巡检 → `run_sell_pass` 盘中卖出巡检 |
| ≥ 15:05 | `CLOSE_BATCH`（一次） | `Engine.close_batch` (`main.py:1207`) | 收盘全量快照 → flush 落盘 → 对账（必要时补采重对）→ 对账失败置次日阻断标记 → 清理过期台账 |
| ≥ 15:35 | `SYNC`（一次） | `RemoteSyncJob.run` (`sync_job.py:61`) | 本机 SQLite → 远端信号侧 HTTP 幂等回流（`synced=0` 行逐行 POST，失败下轮补） |

> **关键时点边界**（`scheduler.py:38-42`）：`_PREMARKET_START=08:55` / `_AUCTION_START=09:15` / `_INTRADAY_START=09:30` / `_DEFAULT_CLOSE=15:05` / `_DEFAULT_SYNC=15:35`。

**当前建仓时点口径（有意设计）**：只打「竞价 / 一字封板」一类——竞价 + 定盘窗（9:15–9:30）的因子轮询定夺，9:30 退出，**不做开盘后连续段（10:30 才封板那种）的盘中追买**；卖出（止损/炸板/尾盘）则覆盖盘中，由 `QMT_SELL_PASS_LIVE` 门控。

### 2.2 关键状态机枚举

- **`OrderState`**（下单台账，`enums.py:45`）：`PLANNED → SUBMITTED → REPORTED → PART_TRADED → TRADED`；撤单 `CANCELLING → CANCELLED`；`REJECTED`/`ERROR`。终态集 `terminal()`、活跃集 `active()` 用于幂等判定。
- **`PositionState`**（持仓，`enums.py:116`）：`LOCKED_T1`（当日锁定）→ `HOLDING` → `SELLING` → `SOLD`/`PART_SOLD`；`FROZEN` 为保留态（当前无赋值点）。
- **`PositionMode`**（续持模式，`enums.py:131`）：`SIGNAL_DRIVEN`（次日又涨停，吃先验续持）/ `TECH_EXIT`（纯技术退出，仅盘口驱动）。
- **`AuctionPhase`**（竞价时段，`enums.py:74`）：`PRE_AUCTION`(<9:15) / `AUCTION_CANCELABLE`(9:15–9:20) / `AUCTION_LOCKED`(9:20–9:25 不可撤) / `SETTLED`(≥9:25 定盘) / `CLOSED_WINDOW`(≥9:30)。
- **`RiskVerdict`**（风控裁决，`enums.py:146`）：`ALLOW` / `FREEZE`（冻结一切新决策）/ `SELL_ONLY_HOLD`（空仓闸门，只锁买不锁卖）。
- **`TradeSide.UNKNOWN`**（`enums.py:27`）：方向不可判定的显式未知态——**绝不臆造成 BUY**，落库留痕，持仓更新对 UNKNOWN 拒绝改仓 + 强告警。

## 3. 盘前装载（watchlist → 可交易名单）

入口 `WatchlistLoader.load(today)`（`watchlist_loader.py:70`），产出常驻内存的 `WatchlistContext`（可交易名单 / 观察名单 / 开仓权限）。盘中下单只读这个内存字典，**不再跨网络**。

### 3.1 取数：盘前 prefetch + 盘中本机读

- **盘前 prefetch**（`remote_watchlist.py:169` `WatchlistPrefetcher.prefetch`）：日历反推 `signal_T = prev_open(today)` → HTTP `GET /api/internal/watchlist?date=signal_T` → 字段映射 `LimitUpWatchlistItem → SelectedStockRow`（唯一映射点）→ 落本机 SQLite `watchlist` 表（latest-wins：先 DELETE 当日再 INSERT OR REPLACE，单事务）。
- **盘中装载来源**（`QMT_WATCHLIST_SOURCE`）：生产走 `SqliteSelectedStockSource`（读本机表，`watchlist_source.py:48`）；另有 `DbSelectedStockSource`（直读 MySQL）与 `HttpSelectedStockSource`（当前为死代码，生产不用）。
- **prefetch 返回码**（`remote_watchlist.py:16`）：`>0` 成功 N 行 / `0` 合法空名单（**不重试**，避免空仓日反复重连）/ `-1` 真失败（调度在 9:15 前重试）。

### 3.2 逐行过滤（五道闸，单票异常隔离）

`_split_by_tradable`（`watchlist_loader.py:199`）对每行依次判：

1. **代码归一**（`identity.resolve_code`）：`SH600000`/`600000`/`600000.SH` → 标准 `600000.SH`；归一失败 → 观察名单。
2. **Universe 前缀白名单**：仅放行 `600/601/603/605/000/001/002/003/300/301`（主板 + 创业板）；科创 688/689、北交 8xx、B 股等 → 观察名单。
3. **买入前置过滤**（`buy_prefilter.evaluate`）：ST/退市整理、四板及以上 → 观察名单（详见 §10）。
4. **价位预算**（`board_rules.budget_prices`）：算涨停价/合理开仓区间；`price_source==MISSING`（信号价位与 `signal_close` 全缺，或 `signal_close<=0`）→ 观察名单。
5. **tradable_flag**：为 False → 观察名单；全部通过 → 进可交易名单（key=归一代码，盘中 O(1) 查）。

> 单票脏数据抛错只丢该票 + 告警，**整批继续**（`watchlist_loader.py:217`），绝不因一只票拖垮整批装载。

### 3.3 开仓权限闸门

`_resolve_open_gate(market_state)`（`watchlist_loader.py:335`）取行集首个非空 `market_state` 决定**是否允许开新仓**：

- `market_state == None` → **False**（保守禁开）。
- `market_state ∈ settings.market_state_block`（默认含 空仓/谨慎参与/退潮/冰点）→ **False**。
- 否则（参与）→ True。

该闸门**只锁买、不锁卖**——持仓按既定纪律处置、T+1 延续。

### 3.4 兜底（装载侧 fail-closed 总表）

| 场景 | 行为 |
|---|---|
| 非交易日 | `is_open=False`，禁开新仓 |
| 主备取数均失败 / 非约定异常 | `degraded=True`，禁开新仓 + 告警，**不抛出**（不拖垮常驻进程） |
| 本机库读空（prefetch 失败/空名单） | 无可交易计划行 → 盘中无票可下单 → 只守仓 |
| market_state 缺失 | 保守禁开新仓 |

## 4. 集合竞价四因子与轮询

竞价时窗硬编码于 `common/auction_window.py`（交易所固定结构，不可配）；因子计算 `auction/auction_factors.py`；轮询 `auction/auction_poller.py`；取数 `auction/tick_source.py`。**价位一律 `Decimal`，浮点先 `str()` 再入 Decimal**（防二进制误差）。

### 4.1 四因子（任一字段缺失 → 该因子 None + 标数据质量码，绝不臆造）

| 因子 | 函数 | 口径 | 缺失码 |
|---|---|---|---|
| **① 竞价高开幅度** `open_pct` | `auction_factors.py:63` | `(lastPrice − lastClose) / lastClose` | `NO_PRE_CLOSE` |
| **② 竞价量能比** `auction_vol_ratio` | `auction_factors.py:76` | 竞价段累计撮合量 ÷ 首板**全天**成交量；**时段不对等**（结构性偏低 ~1–5%），**仅作相对强弱加分，回测标定前不作硬弃门槛** | `NO_BASE_VOL` |
| **③ 分时重心 + 趋势** `auction_centroid`/`centroid_trend` | `auction_factors.py:101` | 帧序列按相邻量增量加权均价；末帧价 > 重心→`UP`，< →`DOWN`，否则 `FLAT`。识别「先拉高再砸」诱多 | — |
| **④ 虚拟封单** `SealInfo` | `auction_factors.py:159` | 达涨停时 `买一量×买一价` = 封单额；`封单额 / 流通市值` = 封流比；五档取 best 档 `[0]` | `NO_SEAL_VOL` |

> **量纲待实测**：买一量 `bidVol` 的「手/股」量纲（是否 ×100）需国金真机确认，故 `QMT_SEAL_RATIO_MIN` 默认 `0`（关闭封流比护栏），避免量纲偏差把真实强板误杀在真金买入路径上。

### 4.2 时窗与轮询

- **时段判定**（`auction_window.py:31` `resolve_phase`）：UTC → 东八区时分秒比较；`is_cancel_forbidden`（9:20–9:25 禁撤）、`is_lunch_break`（11:30–13:00）供撤单/卖出巡检跳过。
- **轮询节奏**（`auction_poller.py`）：基础间隔 `QMT_AUCTION_POLL_INTERVAL_SEC`（默认 3s）；**临近 9:20 / 9:25 前 30 秒加密到 ≤1s**（`_next_interval`，防瞬时丢帧）。9:30 进入 `CLOSED_WINDOW 立即退出。
- **两遍处理**（`auction_poller.py:131`，防污染重心）：第一遍纯算 + 缓冲历史（**降级帧 tick=None 不入历史**），第一遍末统一提交历史，第二遍逐票 push 给路由（**单票 push 失败就地隔离、不阻断其它票**）。

### 4.3 竞价择时门控（双开关 fail-closed）

- `QMT_AUCTION_TIMING_ENABLED`（默认 False）：False 时竞价段**只采集不下单**；建仓真正落单仅来自开盘后（OPENING）决策。
- `QMT_AUCTION_TIMING_VERIFIED`（默认 False）：实测放行标记。`assert_safe_to_trade`（`settings.py:375`）启动期强制：`ENABLED=true` 但 `VERIFIED≠true` → **抛 `RuntimeError` 拒绝启动**，杜绝「未实测就开竞价真金下单」。

### 4.4 降级路径

- **降级 A（整体取数失败）**：`get_full_tick` 抛 `TickSourceError` → 本轮 `tick={}`、产空壳快照标 `NO_TICK` → **上报行情健康 `False`** → Engine `_market_feed_ok=False` → 风控 `FREEZE`（安全默认），进程不崩、留痕 `auction_poll_degraded`。
- **降级 B（单票/早期无 tick）**：9:20 前单帧 `NO_TICK` → 策略 `defer`（推迟、不留痕、不锁幂等，等后续帧）；9:20 及以后仍 `NO_TICK` → 竞价强开追改判开盘后单（涨停价/合理上沿均缺则 SKIP）。

## 5. 建仓决策：路由 + 五策略

路由 `entry/entry_router.py`，五策略 `entry/strategies/`。路由**只出决策不下单**。

### 5.1 路由主流程（`EntryRouter.route`，`entry_router.py:142`）

```
on_auction_snapshot(plan, snap)
 ├─ 幂等：ts_code 已决过有效 BUY → 跳过（SKIP 不锁，允许因子转强后重评；release() 可解锁）
 ├─ 第一道闸 _should_skip：
 │    买入前置过滤(ST/四板) → SKIP；market_state 禁开/缺失 → SKIP；tradable_flag=False → SKIP
 ├─ 选策略 _select_action：按 (strategy_family, setup) 关键词路由（英文枚举 DABAN/BANLU/DIXI 先归一为中文）
 ├─ 第二道闸（仅追买族）：open_pct≤0 且 未顶板 且 重心非 UP → 因子全面走弱 SKIP
 ├─ 策略 decide → StrategyOutcome（defer 则 return None 不留痕）
 └─ 组装 EntryDecision（注入 plan_volume / order_phase / 因子快照 / 禁买锚定字段）
```

路由表（`_select_action`，`entry_router.py:245`，未匹配任何战法 → SKIP，不臆造动作）：

| 命中关键词 | 选中策略 |
|---|---|
| 低吸 / 均线 / 回踩（非龙） | `DIP_BUY_MA`（均线低吸） |
| 龙回头 / 龙+高位回踩/分歧 | `LEADER_PULLBACK`（龙回头） |
| (打板/连板/半路) + (竞价/首板) | `CHASE_AUCTION_STRONG`（竞价强开追） |
| 其它 打板/连板/半路 | `CHASE_LIMIT_UP`（打板跟买） |

### 5.2 五策略触发条件与价位口径

| 策略 | 触发条件（核心阈值） | 限价口径 | 下单时段 |
|---|---|---|---|
| **打板跟买** `chase_limit_up.py` | **已顶板** `is_limit_up`（否则弃，`:33`）；封流比 ≥ `seal_ratio_min`（**仅封流比有值时**校验，缺值不否决，`:43`）；涨停价有值；先验闸放行（强度/续板**有值且低于阈值才弃，缺值不拦**，见注②） | `plan.limit_up_price`（涨停价），缺失→SKIP | 竞价即封→AUCTION；定盘后顶板→OPENING |
| **竞价强开追** `chase_auction_strong.py` | **强开 `open_pct ≥ overheat_pct`**（达强开档=到合理上沿之上，**未达即弃**，`:118`，见注①）；**越竞越强 `centroid_trend=UP`**（FLAT 也弃，`:134`）；弱开 `< abandon_pct` / 走低 `DOWN` / `open_pct=None` → SKIP；量能比弱只留痕不弃；先验同上 | `last_price` → `reasonable_open_high` → `limit_up_price`，以涨停价封顶 | 定盘前 AUCTION；定盘后/降级B OPENING |
| **均线低吸** `dip_buy_ma.py` | **低吸区间 `auction_lowbuy_pct_low/high` 必须显式配**，否则 fail-closed SKIP（`:38`）；`open_pct ∈ [low, high]`；**超 `overheat_pct`（追高）→SKIP**（与 chase 方向相反，见注①）；跌破 `low`→SKIP；先验同打板跟买 | `min(last_price[脏帧≥下沿×0.9守卫], reasonable_open_low)`（不追价、取低） | 固定 OPENING |
| **龙回头** `leader_pullback.py` | 龙头强度 ≥ 下限（**缺失也→SKIP**，`:39`）；续板概率 ≥ 0.3（**缺失也→SKIP**，`:47`）；不看开盘档、纯靠先验（比追板/低吸严，见注②） | `last_price` → `reasonable_open_low` | 固定 OPENING |
| **放弃** `skip.py` | 命中任一放弃条件的统一出口 | 无（仍留痕供归因） | — |

> **量能比已弱化**（评审 F3）：竞价强开追不再因量能比低硬弃，只留痕 `weak_vol`，由 `open_pct + centroid_trend` 等可信因子定买弃。
>
> **注①（`overheat_pct` 一值两用，最易误解）**：`overheat_pct`（缺省回退合理上沿折算，`base.py:111/127`）是「高开线」。**追买的 chase-strong 要 `open_pct ≥ overheat`（达此线才算强开、买）；低吸的 dip 要 `open_pct ≤ overheat`（超此线=追高、弃）**——同一阈值在两族里方向相反，勿混。
>
> **注②（先验闸两档严格度）**：`chase_limit_up` / `dip_buy_ma` 走共享 `prior_gate_reason`（`base.py:144`）——龙头强度 / 续板概率**有值且低于阈值才弃，缺值放行**；`leader_pullback`（`leader_pullback.py:39/47`）自带内联判定——**强度 / 续板缺值也弃**（更严，因它纯靠先验入场）。

### 5.3 仓位数量 `plan_volume`

路由层**不自算仓位**：`_build_decision`（`entry_router.py:289`）仅当 action≠SKIP、注入了 `position_sizer` 回调且 `limit_price` 有值时，调回调产出 `plan_volume`（**强度加权资金分配在此回调注入**，口径见 §11）；否则 `plan_volume=None` 交下游下单层处理。不足 1 手由下单层/sizer 处理。

### 5.4 策略选择直觉：两族方向、典型盘面、当前默认净效果

**两族读法相反（同一组因子，方向相反）**：追买族（`CHASE_*`）追高——要高开 / 顶板 / 越竞越强，因子全面走弱即弃（§5.1 第二道闸）；低吸回踩族（`DIP_BUY_MA` / `LEADER_PULLBACK`）反着来——要平开 / 微跌回踩，**不**走「全面走弱」闸（走弱恰是它们的合法入场）。

| | 追买族 `CHASE_*` | 低吸回踩族 `DIP` / `LEADER` |
|---|---|---|
| 开盘要求 | **高开**（chase-strong 要 `open_pct ≥ overheat`） | **平开 / 微跌**（dip 要落 `[lowbuy_low, high]`，常含负数） |
| 因子走向 | 高开 → 顶板 → 封单稳 / 越竞越强 UP | 低开回踩 → 承接 |
| §5.1 走弱闸 | 触发即弃 | 不受此闸 |
| 主导信号 | 实时盘口 | 信号侧先验（dip 另加低吸档） |

**典型盘面三例**：

- **竞价一字封板的龙头**（family=打板 / 连板接力）→ `CHASE_LIMIT_UP`：`is_limit_up=True` + 涨停价有值 + 先验放行 → 判买、挂涨停价排队（封流比闸默认关，不被封单卡）。
- **高开放量越竞越强的跟风**（family=打板 / 首板 + 竞价）→ `CHASE_AUCTION_STRONG`：`open_pct ≥ overheat` + `centroid_trend=UP` → 判买（末帧价封顶涨停）；末帧 `DOWN`（拉高又砸）判诱多弃；9:20 后竞价取不到 tick → 降级改判 OPENING 开盘后追。
- **平开微跌回踩首板**（family=低吸 / 均线粘合）→ `DIP_BUY_MA`：低吸档已配且 `open_pct∈[low,high]` → 判买（挂 `min(末帧价, 下沿)`）；低吸档未配 → fail-closed 弃。

**当前默认配置下「实际谁会出手」**（仅指策略层；与 §4.3 竞价择时门控、§F 安全门控叠加后的净效果）：

- `QMT_SEAL_RATIO_MIN=0`（§4.1）→ `CHASE_LIMIT_UP` 对封单大小不卡控，**任何顶板无条件跟买**（防炸板护栏待目标机实测 `bidVol` 量纲后配正值如 `0.005` 才生效）。
- 低吸档 `QMT_AUCTION_LOWBUY_PCT_LOW/HIGH` 未配 → `DIP_BUY_MA` **整族 fail-closed 不出手**。
- `QMT_AUCTION_TIMING_ENABLED=False`（§4.3）→ 竞价段只采集，`CHASE_AUCTION_STRONG` 的「9:20 前竞价单」路径不真正下单，限于采集 / 降级后开盘追。
- `QMT_AUCTION_ABANDON_PCT / OVERHEAT_PCT` 默认 None → 强弱开判定回退用信号侧合理高开上 / 下沿折算（`base.py:122/127`），**实际依赖信号侧参考价**。
- `QMT_LEADER_STRENGTH_MIN` 默认 None → 强度闸基本失效，追买族全靠续板≥0.3（有值时）与盘口把关。
- `market_state` 缺失 / 谨慎参与 / 退潮 / 冰点 → 全策略禁开新仓（§3.3 总阀）。

> **策略层净结论**：当前默认配置下，真正「实时盘口验证后会出手」的主力是 `CHASE_LIMIT_UP`（顶板跟买、不被封单闸卡）与 `LEADER_PULLBACK`（靠先验承接）；`CHASE_AUCTION_STRONG` 因竞价择时关而限于采集 / 降级开盘追；`DIP_BUY_MA` 因低吸档未配而当前不出手。能否落成真实委托还需过 §6 下单层闸（`KILL_SWITCH` 等）。

### 5.5 打板因子消费（watchlist 1.2.0 信号侧因子，执行侧 E2，默认即生效·起步值）

信号侧 watchlist 契约 1.2.0 下发 6 个打板因子，执行侧经 E1 透传到 `PlanRow`（封板时序 `first_limit_time`/`last_limit_time`/`open_times` + 位置/强度 `volume_ratio`/`return_5d_pct`/`return_10d_pct`）。E2 让策略消费其中 3 个，新增弃条件（**默认即生效·起步值**，env 可覆盖；统一双守卫「settings 阈值与 plan 因子均非 None 才判弃」缺数据零误杀；时刻经 `base._parse_hms` 脏/缺 fail-open）：

| 因子 | 阈值（§E） | 语义 | 挂载策略 |
|---|---|---|---|
| `open_times` | `QMT_FORBID_OPEN_TIMES_MAX` | `>=` 本值→弃（反复炸板/烂板） | `chase_limit_up` / `dip_buy_ma` / `leader_pullback` |
| `return_5d_pct` | `QMT_HIGH_RETURN_PCT_LIMIT` | `>=` 本值→弃（高位规避，手册首要风险闸） | 追买族 `chase_limit_up` / `chase_auction_strong`（放降级 B 前）；低吸/龙回头（回踩战法）不接 |
| `first_limit_time` | `QMT_PULLBACK_ENTRY_DEADLINE_HM`(HH:MM) | `>` 本值→弃（首封太晚） | 仅 `leader_pullback` |

> **现状**：3 阈值**默认即生效**起步值 `1`(只打稳封) / `40`(涨超40%规避) / `10:00`（业务方决策 2026-06-19；env 可覆盖、关停配永不触发的大值，⚠️`open_times` 勿配 0=全弃；双守卫缺数据零误杀），为**未回测的起步值**待标定优化。`last_limit_time`/`volume_ratio`/`return_10d_pct` 仅透传留痕、不接判定（`volume_ratio` 是信号侧日线量比，**≠** §4.1 盘中 `auction_vol_ratio`，严格区分不混用）。详见 [doc/23 §四](23-打板因子补充-watchlist契约1.2.0-设计与台账.md)。

## 6. 下单执行：唯一下单点 + 三层幂等 + TTL + 一字秒封

`order/order_executor.py` 是**唯一下单出口**（所有买卖、转次优都过它），本地台账 `order/local_ledger.py`。**绝不自判成交**：本类最多推到 `SUBMITTED`，成交由外部回报 `add_fill` 驱动。

### 6.1 下单主流程（`place`，`order_executor.py:301`）— 9 道前置闸

1. SKIP 决策 → 只留痕不下单。
2. **买入禁买硬规则**（`buy_prefilter.evaluate`，ST/四板）→ 拒单。
3. **`kill_switch`** → 只采集不下单。
4. **业务单号幂等**：`ledger.find_active` 命中活跃单 → 返回既有单号，不重复下单。
5. **单日下单次数上限** `max_orders_per_day`（含转次优/卖出）。
6. **单日建仓只数上限** `max_positions_per_day`（按 ts_code 去重，占名额口径见下）。
7. **计划股数**：`plan_volume` 缺失且 `allow_plan_volume_fallback=false` → fail-closed 拒单；含资金/单票/单笔上限校验。
8. **限价合法性**：`≤0` 或超涨停价 / 偏离护栏（`price_deviation_guard_pct`）→ 拒。
9. **发单前关键落盘** `_persist_critical_before_order`：写线程死/commit 失败 → fail-closed 拒发；纯超时 → 强告警后继续。

通过后：生成业务单号 + order_remark → 落台账 `PLANNED` → 同步落盘 → `trader.order_stock` → 推 `SUBMITTED` + 回填 `order_id` → 再同步落盘 order_id（堵「券商有单、台账只认 PLANNED」孤儿窗口）→ 计算 TTL 截止、占下单次数。

### 6.2 三层幂等

| 层 | 机制 | 防什么 |
|---|---|---|
| **① 业务单号** `biz_order_no` | `{date}_{ts_code}_{family}_{seq:03d}`，按维度内存自增 | 可读唯一键，透传对账 |
| **② 台账活跃单复核** `find_active` | `(date,ts_code,family)` 活跃索引（超集只增不删）+ 按权威态 `state∈active()` 复核 | 重复推送 / 断线补发 / 重启重放 → 不重复下单 |
| **③ 成交编号** `traded_id` | `add_fill` 按 `traded_id` 去重（无 id 用 `order_id+vol+price` 兜底键），存 `counted_trade_ids` | 回报重投 / 重连 → 不重复累计成交 |

重启重建：`load_from_db` 读回台账 → `rebuild_seq_counter`（按最大 seq 续号）、`rebuild_runtime_state`（TTL + 下单次数）、`reconcile_fills_from_detail`（按成交明细重算 `filled_volume`+去重集）。`_order_index` 两级映射 `(order_id→{date→biz})` 防 order_id 跨日复用串单。

### 6.3 TTL 撤单（`_compute_ttl_deadline` / `sweep_expired` / `on_ttl_expired`）

- **TTL 语义**：竞价单存活到 **9:25 定盘**；开盘单 `now + order_ttl_seconds`（默认 60s）。TTL 存内存（不污染台账契约）。
- **巡检跳过**：`9:20–9:25 禁撤段` + `午休` 整体跳过（撤单必废，避免废弃一字板唯一单）。
- **到期处理**：在途单 → `_safe_cancel`（异常隔离，失败续宽限期重试不打断整轮）→ 受理成功置 `CANCELLING`；若买单零成交 → 推断 `miss_reason` + 发 `BUY_MISS` 决策 + 转次优。
- **撤单回执丢失兜底**：`CANCELLING` 单超 `cancel_grace_seconds`（默认 30s）→ **幂等重发 cancel**（券商无此活单会自拒），防永久卡 `CANCELLING` 占名额占预算。

### 6.4 一字秒封 / 买不进

- `_infer_miss_reason`：`is_one_word`→「一字未成」、`is_second_seal`/`is_limit_up`→「秒封未成」、其它→「排队未成」。
- 买不进（TTL 到期零成交）→ 标 miss_reason + `try_next_best`（转次优链，递归承接）；次优为空或仍失败 → 仅留痕，**不追高破板**。
- **建仓名额占用口径**（`_is_position_committed`）：有成交（含部成）→ 占；零成交但在途（PLANNED~CANCELLING）→ 占（结果未定保守占位）；**终态零成交（一字/秒封买不进、拒单、错误）→ 释放名额**。

### 6.5 台账状态机与异常兜底

`OrderState`：`PLANNED→SUBMITTED→REPORTED→PART_TRADED→TRADED`；撤单 `CANCELLING→CANCELLED`；`REJECTED`/`ERROR`。部成口径：`CANCELLED/REJECTED` 但 `filled>0` → 收口为 `PART_TRADED`（已成部分是真实建仓）。

| 异常 | 兜底 |
|---|---|
| 同步下单失败（order_id<0/None） | 置 `ERROR` + 上报通道不健康 + 转次优（不放弃整批） |
| 下单通道异常 | `_report_conn(False)` → Engine 心跳累计达阈值置 FREEZE |
| 发单前落盘确定性故障 | fail-closed 拒发（堵重复下单窗口） |
| 发单后落盘超时 | 委托已发不可回退，仅强告警 |
| 成交回报无对应 order_id | `add_fill` 忽略（防越权改写），人工补账 |

## 7. 持仓与卖出

持仓 `position/position_manager.py`，卖出决策 `position/sell_decider.py`，卖出簿 `position/sell_book_builder.py`，下卖单编排 `main.py:run_sell_pass`。

### 7.1 T+1 双保险

- **业务闸 `earliest_sellable_date`**：首建仓 `= calendar.next_open(buy_date)`（**走交易日历，禁自然日 +1**，跨周末/节假日才对）；加仓取 min。`sellable_units` 只返回 `today ≥ earliest_sellable_date` 的单元。
- **量闸 `can_use_volume`**：当日买入初始 0（`LOCKED_T1`），跨可卖日 `refresh_state` 置 `=volume`。可卖上限 `= max(0, can_use_volume − on_road_sell_volume)`（扣在途未成卖量，防超挂）。

### 7.2 卖出触发条件

前置短路（`_pre_guard`，优先级最高）：`LOCKED_T1`→HOLD（守 T+1，连决策都不进）；风控 `FREEZE`→HOLD；`SELL_ONLY_HOLD`→放行（空仓闸门不锁卖）。

| 通道 | 触发 | 动作 |
|---|---|---|
| **单票浮亏止损**（最优先，`main.py:_unit_stop_loss_breached`） | `(avg_cost − last)/avg_cost ≥ stock_float_loss_limit` | CLEAR |
| **竞价定夺**（9:15–9:25，`decide_auction`） | 盘口整体缺失→HOLD（安全默认）；量价背离 / 弱开（`open_pct≤0`）→ REDUCE/CLEAR；高开强+量匹配+先验强→HOLD | — |
| **分时定夺**（9:30+，`decide_intraday`） | 炸板→CLEAR；破位（破位+放量）→CLEAR；烂板（开板≥2次）→CLEAR；秒板稳封→HOLD；冲高量价透支→REDUCE/CLEAR；尾盘弱→REDUCE/CLEAR | — |

### 7.3 先验 / 纯技术分叉

- `refresh_state` 每日跨可卖日取先验：次日又涨停（命中信号侧 watchlist）→ `mode=SIGNAL_DRIVEN`（吃续板先验）；否则 → `mode=TECH_EXIT`（纯盘口驱动）。
- `_is_prior_strong`：`TECH_EXIT` 或 `prior=None` 或 `continuation_prob` 缺失/`< continuation_high`（默认 0.6）→ 判**弱**。
- **影响**：盘口触发减仓信号时，先验强 → `REDUCE`（保底仓博续板）；先验弱 → `CLEAR`（缺先验不恋战，保守全出）。

### 7.4 卖出价位与卖量钳制

- **价位**（`_place_sell`）：优先实时 `last_price`（>0），缺失回退 `avg_cost`（告警），仍无效（≤0）→ 放弃卖出。不用成本价挂单（炸板时成本远高于现价会挂不出）。
- **卖量**：CLEAR 卖全部可用量（含零股）；REDUCE 按比例（缺省 0.5）**向下取整到 100 股**；再经 `risk.clamp_sell_volume` 钳到 `≤ 可用量`，`≤0` 不发单。

### 7.5 卖出簿与当前限制（重要）

`build_order_book`（`sell_book_builder.py:32`）把单帧 tick + PlanRow 构造成 `OrderBook`。当前最小版（阶段0-C）**只能判单帧字段**（`last_price`/`open_pct`/`is_sealed`）；**跨帧状态量（炸板 `broke_board`、破位 `below_support`、放量 `volume_surge`、量价背离、开板次数、尾盘走弱）全部保守默认 False/0、不参与决策**。

已知风险（待阶段1 T1.2 根治）：① 不在今日 watchlist 的隔夜真封板票无 plan → `is_sealed` 误判 False，可能被误清；② watchlist 取数失败 → 全部 `is_sealed=False` → 批量误清仓。

### 7.6 生产门控 `QMT_SELL_PASS_LIVE`（默认关）

- **关（默认）**：`prior_provider=None`，盘中不自动卖（回退到接线前安全行为）。
- **开**：`assert_safe_to_trade` **强制要求已配 `QMT_STOCK_FLOAT_LOSS_LIMIT`**（`settings.py:383`），否则拒启——因跨帧破位/炸板止损未保真，单帧浮亏止损是唯一价位安全网。**须待 T1.2 跨帧保真 + 隔夜补数 + 真机实测后才开。**

## 8. 风控护栏（`risk/risk.py`）

`Risk.gate` 返回 `RiskDecision`，按优先级裁决：

1. **关键输入不可信 → FREEZE**（最高）：`market_feed_ok=False`（行情断流）或 `trade_conn_ok=False`（下单断线），任一即冻结——没可信盘口/通道宁可不交易。
2. **账户级击穿 → FREEZE**：当日回撤 ≥ `account_drawdown_limit` 或已实现亏损 ≥ `account_loss_limit`。仅在**开仓路径**喂账户回撤；**卖出路径刻意不喂**（避免回撤反而冻结必要的止损出场）。
3. **单票浮亏击穿 → FREEZE**：`unit_float_loss ≥ stock_float_loss_limit`（当前两调用方均喂 None，为预留）。
4. **空仓闸门 → SELL_ONLY_HOLD**：`market_state=="空仓"` → 只守仓不新开、存量可卖。
5. 其余 → **ALLOW**。

辅助口径：

- **阈值击穿** `_breached`：阈值未配（None）→ 不约束；观测值缺失（None）→ 返回 False（不凭空冻结）；用 Decimal 比较，「达到红线即击穿」（≥ 偏保守）。
- **开仓闸** `is_open_blocked`：`market_state ∈ market_state_block` 即禁开（None 不在此兜底冻结，由 gate 关键输入分支负责）。
- **卖量钳制** `clamp_sell_volume`：`min(决策量, 可用量)`，负值兜底 0，绝不下超量卖单（守 T+1 量闸）。
- **冻结留痕**：`_freeze` 落 error 日志（恢复需人工确认再解冻）。

> **日初权益基线 fail-closed**：配了 `account_drawdown_limit` 时，盘前抓基线失败 → 禁开新仓（不冻结卖出，`main.py:585`）。

## 9. 连接守护与重连（`connection/connection_guard.py`）

- **就绪判定**：**只读 `connect()` 返回值**（0 即连上），**不依赖 `on_connected` 回调**（QMT 推荐）；connect 或 subscribe 任一非 0 → `ready=False`，connect 失败不继续 subscribe（绝不进半就绪态）。
- **断线重建**：`on_disconnected` 立刻冻结开仓闸（`ready=False`）→ `reconnect`：从 provider 取**新 `session_id`**（复用旧 session 会订阅失效无回报）；连续吐相同 session 超 `max_session_retry`(5) 次 → fail-closed 拒重连 + 强告警。重连就绪触发一次补采 `on_reconnect_backfill`。
- **健康汇入风控**：`on_connection_state(bool)` 绑到 `Engine.report_trade_conn`（解冻/冻结权威源）；盘中 `trade_conn_heartbeat` 主动探活（`query_stock_asset`），连续失败达 `trade_conn_heartbeat_fail_threshold`(默认3) 才冻结、单次成功清零、首次跨阈值请求重连。
- **session_id 生成**（`run.py:128`）：毫秒低 15 位 + 15 位随机 + 单调自增，收口到正 int32，保证同秒重启也不与历史重叠。

**交易日历**（`common/trade_calendar.py`）：生产用 `StaticTradeCalendar`（读 `QMT_TRADE_CALENDAR_FILE`，含法定节假日，与信号侧 `a_trade_calendar` 同源）；`WeekdayTradeCalendar`（仅排周末）仅离线/测试用。**无日历文件且 `allow_weekday_calendar=false` → 拒绝启动**（`run.py:199`）。

## 10. 三层禁买闸（绝不买 ST / 禁买四板及以上）

单一来源 `common/buy_prefilter.py`：有序规则集 `_RULES=(_rule_st, _rule_high_board)`，`evaluate(view, high_board_min_level)` 命中第一条即返回结构化裁决 `PrefilterVerdict(allowed, rule_code, reason)`。

| 规则 | 判定口径（fail-closed） |
|---|---|
| **RULE_ST** | `is_st_stock(is_st, name)`：信号侧显式 `is_st=True` **或** 证券名含 `ST/*ST/退/*退`（最严格口径，防 is_st 漂移漏判）→ 禁买 |
| **RULE_HIGH_BOARD** | `min_level≤0` 关闭；`board_level>0` 时 `≥min_level` 禁买（精确口径，默认 4=四板及以上）；**`board_level` 缺失但 `tier==HIGH_BOARD` → 禁买（fail-closed，按 4+ 板保守拦）**；双源无证据 → 放行（绝不无证据全拦） |

**三层叠加**（同口径 `is_st_stock` / `is_high_board`，三处一致）：① 盘前装载剔出可交易名单 → ② 建仓路由 `_should_skip` SKIP → ③ 唯一下单点 `place` 据 `EntryDecision.is_st/board_level/tier` 最终拒单。`is_st`/`board_level`/`tier` 经 HTTP→SQLite→PlanRow→EntryDecision 同路透传。

辅助（`universe_filter.py` / `board_rules.py` / `identity.py`）：

- **Universe**：前缀白名单（10 段主板+创业板）+ 停牌/退市/ST 名称综合校验，非白名单段转观察名单。
- **板块规则**：主板 ±10%（ST 主板 5%）、创业板 ±20%；涨停价 `round_to_cent(close×(1+ratio))`；信号侧给价优先，缺失才按 board 现算，board 判不出且无信号价 → `MISSING` 转观察名单（不按主板 10% 误算）。
- **代码归一** `resolve_code`：后缀只认规范位置（末尾 `.SH/.SZ/.BJ` 或开头 `SH/SZ/BJ`），显式后缀与前缀推断矛盾 → None（脏数据，禁交易）。

## 11. 资金分配口径（强度加权 + 只数上限）

有限资金 + 多候选：**先按强度优选 top-N 进买入名额，再在 N 只内按 `leader_strength_score` 强度加权分配**（强的多分、名额内满额部署不闲置现金）。

- **单日只数上限** `QMT_MAX_POSITIONS_PER_DAY`（默认 5）：盘前按强度降序取 top-5 进 universe，权重 `w_i = 强度_i / Σ(top-5 强度)`（Σw=1）。其余候选预算 0、不开仓。下单层叠加只数硬闸（占名额口径见 §6.4）双保险。
- **每票计划金额** `= min(可用现金, 总预算上限−已承诺, 总预算上限×w, 单笔上限, 单票上限)` ÷ 限价向下取整到 100 股；不足 1 手不下单。权重盘前一次算定、与触发时序无关（弱票先达标也只吃自己份额）。
- **总预算上限** `= 日初权益 × QMT_TARGET_POSITION_RATIO`（默认 1.0），与 `QMT_MAX_TOTAL_EXPOSURE` 取小。
- **已承诺** = 当日活跃买单（在途+已成）金额和，下单前扣减（防 `frozen_cash` 未及时刷新时多单各按全额测算而超额废单）。
- 继续持有的持仓已是 `market_value` 占用、不在可用现金里；不卖不回笼，新票只用剩余现金。

## 12. 回流落库 / 异步持久化 / 盘后同步 / 对账

### 12.1 本地化数据栈（单进程 + SQLite，write-behind 绝不阻塞交易）

`storage/write_queue.py` 单写线程 + WAL：

- `submit` 仅 `put_nowait`（交易线程永不等磁盘）；写线程内异常只吞不传播（写失败不影响交易）。
- **有界溢出熔断** `QMT_WRITE_QUEUE_MAX`（默认 5万，0=关）：积压超限丢任务 + 告警 + `on_failure`（防写线程挂死无界堆积 OOM）。
- **连续 commit 失败熔断**（默认 5 次）+ **卡死看门狗** `QMT_WRITE_QUEUE_STUCK_SECONDS`（默认 30s，0=关）：有积压却连续无推进 → `is_healthy=False` → fail-closed 停开新仓。
- **关键落盘确认** `flush_confirm`：屏障任务读 `_batch_ok`，区分「全 commit 成功 / 有失败行」，供发单前 fail-closed 校验（写线程死直接判失败不等超时）。

### 12.2 回流四表（`data_writer/`）

`qmt_trade`（成交）/ `qmt_order`（委托）/ `qmt_position_snapshot`（持仓快照，OPEN/INTRADAY/CLOSE）/ `qmt_account_daily`（账户日快照）。

- **幂等 upsert**：`INSERT ... ON DUPLICATE KEY UPDATE`；普通列后到覆盖，`signal_trade_date`/`*_east8` 走 **COALESCE（空不覆盖）**；唯一键列不进 UPDATE。
- **唯一键加固** `QMT_UNIQUE_WITH_TRADE_DATE`（默认 true）：明细唯一键纳入 `trade_date`，防 order_id/traded_id 跨日复用串号。
- 成交/委托是唯一事实源，落库失败**重抛**绝不静默丢；快照按 `snapshot_type` 互不覆盖（历史净值只认 CLOSE）。

### 12.3 盘后同步（`storage/sync_job.py`）

读本机 `synced=0` 行 → 逐行 POST `/api/internal/qmt/ingest`（单条幂等，**一行失败不中断整表、保 `synced=0` 下轮补**）→ 成功异步标 `synced=1`（`WHERE synced=0` 守卫，防把未同步的最新变更误标已同步）。打板单日回流量小，逐行往返开销可忽略。

### 12.4 对账（`reconcile/reconcile.py`，四类勾稽）

| 类 | 对什么 | 偏差响应 |
|---|---|---|
| **委托对账** | 台账计划单 vs `qmt_order` 回报（强关联 order_id、弱关联 remark/方向） | `missing_report`(漏单,error) / `order_failed`(warn) / `manual_order`(手工单,warn) |
| **成交对账** | `qmt_order.traded_volume` vs Σ`qmt_trade` | 不勾稽 → `needs_backfill` 触发补采重对 |
| **资产对账** | Σ成交净额(买负卖正) vs 账户现金变动 | 偏差 > 容差 = `max(QMT_RECONCILE_ASSET_ABS_FLOOR 默认1000, 成交额×QMT_RECONCILE_ASSET_REL_RATE 默认0.003)`（覆盖佣金/印花税噪声）→ **告警不阻断** |
| **滑点对账** | 计划价 vs 实际成交均价 | 指标留痕供复盘 |

- `signal_trade_date` 回填：优先 order_remark 解析（权威），缺失才 `calendar.prev_open(trade_date)` 反推（禁自然日 -1）。
- **对账系统本身不阻断开仓**（仅告警）；但 `close_batch` 对账真异常（漏单/手工单/成交不勾稽）→ 置**持久阻断标记** → 次日 `prewarm` 据此**只守仓不开新仓**（人工清除前持续）。

### 12.5 HTTP 客户端（`common/http_client.py`）

`X-Internal-Token` 头鉴权（不入日志）；超时 `QMT_HTTP_TIMEOUT_SECONDS`（默认 10s）；非 2xx/网络/解析失败统一抛 `SignalHttpError`（body 截 500 字符防泄露）。按接口分 token：`resolve_watchlist_token` / `resolve_ingest_token`，缺省回落统一 `signal_internal_token`（文件优先于环境变量）。

## 13. 意外情况兜底逻辑总览（fail-closed 全景）

> 总原则：**不确定时一律降级「只守仓、不开新仓」；卖出能力尽量保留（行情/通道断时才冻结卖出）。**

| 场景 | 检测点 | 兜底动作 |
|---|---|---|
| watchlist 拉取失败 / 名单为空 | `WatchlistLoader.load` | 降级 `degraded`，禁开新仓，不抛错不崩进程 |
| 非交易日 | `calendar.is_open` | 空载上下文，禁开新仓 |
| market_state 缺失 / 禁开集合 | `_resolve_open_gate` / `risk` | 禁开新仓（空仓闸门只锁买） |
| 行情整体断流 | `AuctionPoller` 降级 A / 盘中取盘口失败 | 上报 `market_feed_ok=False` → 风控 FREEZE |
| 单票/早期无 tick | 降级 B | 早期 `defer` 等后续帧；9:20 后改判开盘后单或 SKIP |
| 竞价数据字段缺失 | 四因子 | 该因子 None + 数据质量码，不臆造、不据缺失加分 |
| 低吸区间未配 | `dip_buy_ma` | fail-closed SKIP（绝不用高开区间臆造低吸档） |
| `plan_volume` 缺失 | `order_executor.place` | fail-closed 拒单（除非显式开 `allow_plan_volume_fallback`） |
| 下单通道异常/同步失败 | `place` / 心跳 | 转次优 + 上报不健康；连续失败达阈值 FREEZE |
| 发单前落盘确定性故障 | `_persist_critical_before_order` | fail-closed 拒发（堵重复下单窗口） |
| 一字/秒封买不进 | `on_ttl_expired` | 标 miss_reason + 转次优；失败仅留痕不追高，终态零成交释放名额 |
| 撤单回执丢失 | `CANCELLING` 宽限 | 超 grace 幂等重发 cancel |
| 9:20–9:25 / 午休 | 时窗判定 | 跳过撤单巡检（撤单必废） |
| 写线程死 / 卡死 / 溢出 | `write_queue` 健康 | `on_failure` → 停开新仓 + 强告警 |
| 日初权益基线抓取失败 | `prewarm`（配了回撤限时） | 禁开新仓（不冻结卖出） |
| 对账真异常 | `close_batch` | 置持久阻断标记 → 次日只守仓（人工清除前） |
| 隔夜真封板票无 plan / watchlist 失败 | `sell_book_builder` 最小版 | 已知风险，靠 `SELL_PASS_LIVE` 默认关 + 单帧浮亏止损兜底，待 T1.2 |
| 连接断线 / session 复用 | `ConnectionGuard` | 冻结开仓闸 → 换新 session 重连；复用超限 fail-closed 拒重连 |
| 无真实交易日历 | `_build_calendar` | 拒绝启动（除非显式 `allow_weekday_calendar`） |
| 方向不可判定 | `TradeSide.UNKNOWN` | 拒绝改持仓 + 强告警，不臆造为 BUY |
