## 0. 本轮最关键的口径纠偏（缺测走偏）

负责人原意：**某只票缺失重要数据时，只是不买这一只票**。评审发现实现走偏成：把 LLM 软先验「续板概率 continuation_prob」也当成了「核心交易指标」，于是大模型一次解析失败 / 漏写某票 / 换个近义词，就会把行情数据齐全的好票（甚至整档）误判为「数据缺测」而弃买。

已修正为：**核心缺测指标只含真正重要的行情/因子数据**（收盘价、连板高度、龙头强度、封流比及分母、量能比及分母、位置收益、封板时序）；continuation_prob 移出缺测集，缺它只落 None、属普通软降级，**该票仍可买**，执行侧按「普通先验缺→按实时盘口把关(fail-open)」消费，真正重要行情数据缺失才弃买该票。并加结构防御：LLM 把先验写成非数组/坏结构时，降级为「无先验」而非崩掉整张当日名单。
（信号-1/4/5/6，commit `d84363e`。另：执行侧 B3「缺测持仓强卖」此前已于 2026-06-21 下线，缺测只在买入侧拦截、不强卖持仓；本轮一并把两仓仍写「缺测强卖」的陈旧注释全部纠正——两系-14。）

## 2. 待决策 / 待实测清单（**未做默认处理，按性质上交**）

### 2.1 必须真机实测才能定（开交易通道前的硬前置）

- **执行-5 / 执行-6（下单方向/价位/状态魔数 23/24/11）**：占位值未在目标机用 `vars(xtconstant)` 实测，且分散在下单侧、入口、回读 normalize 三处（回读是独立副本）。真机值若不符会反转买卖方向或错判委托状态，且改下单侧忘改回读侧会「发一套码收一套码翻译」。**建议**：真机用 `xtconstant` 实测确认全部枚举值并收敛到单一来源常量，再开通道。当前靠通道关 + 熔断压住，未改动。
- **执行-9（弱封减仓以涨停价挂半仓砸自家封板）**：仅当运维把 `QMT_SEAL_RATIO_MIN` 配成正数才触发（默认 0 不触发）；修复涉及「封板状态下减仓如何定价」需真机盘口买卖档数据 + 策略口径定夺（是否封板期不减、或改用可成交价）。**建议**：开该阈值前先在真机定盘口定价口径。未改动。
- **执行-19 关联 / 两系-10（竞价量 volume 单位：股 vs 手）**：量能比分子(实时撮合量)与分母(信号侧 first_board_vol，手)单位可能差 100 倍；当前该因子已只留痕不据此弃单、暂不致错。**建议**：真机确认 QMT tick volume 单位后，与分母对齐（/100 或换同时段竞价量）再把该因子重新当硬门槛。未改动。
- **执行-11（QMT 全推 tick 的 volume 在 9:25 后是否并入开盘量）**：影响定盘段重心因子是否被开盘量污染、进而 OPENING 单买卖方向。**建议**：真机核实 9:25 后 volume 行为后再定是否在定盘段排除该帧增量。未改动。

### 2.2 跨两仓契约 / 口径，需负责人决策（不擅自默认）

- **执行-3（盘中临时实施 ST 漏挡）**：禁 ST 三道闸只看盘前名单的名称/is_st，执行侧无当日实时证券名复核（连字段都没有）。需先研究执行侧 xtdata 是否可取当日实时证券名，再定方案。**建议**：① 信号侧盘前名单尽量用最新 ST 标志；② 调研执行侧实时取名可行性，可行则下单前按实时名复核 ST。属跨仓 + 取数能力问题，交负责人定。未改动。
- **两系-1（回流脏行缺未放宽的 NOT NULL 列→500 死循环）**：放宽名单漏了 trade_side/ts_code/order_status/decision_type/decided_time（决策表整张未进放宽名单）。要让脏行「留痕入库」需迁移把这些列改可空（契约/迁移变更）；要让它「被拒不重传」需配合执行侧死信（见两系-9）。**建议**：与负责人定「这些列哪些可空留痕、哪些应判 422 拒收」，再做迁移。未改动（属迁移+契约决策）。
- **两系-9（执行侧把 4xx 永久失败也当可重试→每天重试都失败永远清不掉）**：需要在本机 SQLite 引入「死信」终态（如 synced=2）+ 区分永久(4xx)与可重试(5xx/网络)失败 + 同步报告 ok 口径是否计死信 + 告警。属 schema + 同步口径变更。**建议**：与两系-1 一并定方案（信号侧坏数据返 422 + 执行侧死信剔除 + 告警）。未改动。
- **两系-12（历史净值 cumulative/年化/回撤未剔出入金，胜率/夏普却剔了→自相矛盾）**：精确 TWR 需可靠的出入金台账（net_cash_flow 当前基本未被赋值）。**建议**：定「是否引入出入金台账做 TWR」或「看板明确标注 cumulative 为含出入金口径」。属口径+数据来源决策。未改动。
- **两系-13（下单成功率把部分成交也算成功，与买不进只数口径不互补）**：属复盘统计口径定义（部分成交算不算成功、未成挂单归哪类）。**建议**：定清三态(成功/买不进/挂着未成)口径再改。未改动（不亏钱、属口径）。

### 2.3 需设计（并发/基础设施），未做脆弱半成品
