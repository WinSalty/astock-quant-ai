## 贯穿性结论（三份评审共同指向）

1. **「数据缺口/接口失败 → 静默退化为 0/旧值/None」是最大系统性风险**：信号侧用 0 或 T-1 旧值冒充并透传执行侧（S-2/S-3/S-9/S-10），执行侧无从感知降级；回流侧脏数据落 None 又撞信号侧 NOT NULL 成毒丸（J-4/5/6）。两端都缺「缺测显式标记 + fail-closed/可对账」的统一口径。
2. **概率/角色/策略语义跨两仓不可比**（J-1/J-2/S-6）：信号侧产出端的「相对档位/降级哨兵/角色置信」被执行侧当绝对阈值/独立字段消费，契约层缺「语义版本对齐」约束。
3. **token 与日期口径依赖运维纪律而非代码自洽**（J-3/J-8/E-5）：两侧不对称之处目前靠文档约定 + 人工配置兜住。
