## 0. 范围与红线

**范围**：执行侧 `qmt_strategy` 上线前需落地的代码任务。分三阶段：

- **阶段 0｜纯离线代码**：不接真机即可全部写完 + fake 单测自证。含**解除唯一硬阻塞的前置**（卖出链最小可用接线）、fail-closed 加固、健壮性加固、死代码/注释清理。
- **阶段 1｜真机实测固化**：依赖国金 miniQMT 登录后才能做。含 §A 占位确认、跨帧盘口字段键名固化、卖出链完整保真（破位/炸板）。
- **阶段 2｜灰度上线**：阶段 0+1 全绿后按硬门槛小仓灰度。

**红线（不可违反）**：

1. **卖出链未接线（且未经 fake 单测自证全分支）前，严禁接实盘** —— 买入即裸奔无止损。
2. 阶段 0 全程**不碰真机**；阶段 1 实测期 `QMT_KILL_SWITCH=true`、竞价择时 `QMT_AUCTION_TIMING_ENABLED=false` 保持关。
3. 触碰任何代码须补/补齐**中文注释**（业务意图 / 边界 / 重跑口径，沿用 `AGENTS.md`）。
4. 占位常量在真机 `vars()` 确认前**不得删除「占位/待实测」注释**；确认后才钉死并去注释。
5. 改动须**全量 `pytest -q` 无回归**；正式交付 commit + push（执行侧 `main`），commit message 末尾带 model 后缀。

## 1. 阶段 0 ｜ 纯离线代码（不接真机即可完成 + 单测自证）

> 目标：把「配好环境也跑不通、必须改代码」的缺口在离线状态下全部补齐并单测自证。完成本阶段 = 解除上线硬阻塞的**代码前置**（剩余保真度依赖阶段 1 真机 tick 键名）。

### T0.1【硬阻塞·最高优先级】卖出链最小可用接线（`sell_books_provider` + 注入）

- **目标**：让生产路径下 `run_sell_pass` 真正可达，先打通「弱开 / 封流比 / 单票浮亏止损」这部分**单帧即可判定**的卖出能力。
- **涉及文件**：`app/run.py`（注入点 `:334`）、新增 `auction/sell_book_source.py`（或 `position/sell_book_builder.py`）、`auction/auction_factors.py`（复用 `_to_decimal`/`_best_level`/`open_pct`/`virtual_seal`）、`contracts/models.py`（`OrderBook`）。
- **做什么**：
  1. 实现 `build_sell_books(tick_source, sellable_codes, plans, clock) -> {ts_code: OrderBook}`：逐 code 调 `get_full_tick` → 用 `auction_factors` 同构逻辑填 **4 个单帧字段**：`last_price`←`tick['lastPrice']`、`open_pct`←`open_pct(lastPrice, lastClose)`、`seal_to_float_ratio`/`seal_amount`←`virtual_seal(tick, limit_up_price, float_mktcap)`。
  2. **7 个跨帧字段（`is_sealed`/`broke_board`/`below_support`/`volume_surge`/`near_close_weak`/`open_times`/`price_volume_diverge`）阶段 0 一律保守置默认**（False/0/None）——此时 decider 走 HOLD/弱开兜底、不会误清仓（真机保真在 T1.2）。
  3. 在 `run.py:main()` 注入 `DailyScheduler(sell_books_provider=lambda today: build_sell_books(...))`；provider 取盘口失败抛异常/返回空的语义已由 `scheduler._run_sell_with_feed_health` 实现（异常→`report_market_feed(False)` 不卖；空集→`True` 但不卖），provider 只需「取不到就抛/返回空、绝不臆造盘口」。
- **验收标准**：
  - 新增 fake 盘口源单测，覆盖 `run_sell_pass` 在 INTRADAY/auction 两 session 下：单票浮亏止损触发 CLEAR、弱开触发 reduce/clear、强开/封流比稳 → HOLD、盘口缺失 → HOLD（不误卖）、provider 抛异常 → feed=False 不卖。
  - 既有 `tests/test_scheduler.py` 的 provider 语义用例回归通过。
  - 生产路径断言：注入后 `scheduler.dispatch(INTRADAY)` 能走到 `engine.run_sell_pass`（用 fake provider 的调度集成测试证明可达）。
- **依赖**：无（可立即开工）。**T1.2 依赖本任务**。

### T0.2 `prior_provider` 续板先验接线（与 T0.1 同批）

- **目标**：消除「持仓恒 `TECH_EXIT` 纯技术退出、续板增强不生效」。
- **涉及文件**：`app/run.py:236`（注入点）、`app/main.py:100`（`EngineDeps.prior_provider` 默认）、`storage/`（读本机当日 watchlist / `limit_up_selected_stock`）、`contracts/models.py`（`SignalPrior`）。
- **做什么**：实现 `prior_provider(ts_code, today) -> Optional[SignalPrior]`，读本机 SQLite 当日名单表（映射 `continuation_prob`/`fail_conditions`/`role`/`strategy`），命中返回 `SignalPrior`、否则 `None`；在 `build_real_engine` 注入 `EngineDeps`。单票取数异常已由 `main.py:939-943` 隔离，provider 取不到返回 None 即可。
- **验收标准**：fake watchlist 源单测覆盖 `SIGNAL_DRIVEN`（先验强→REDUCE 保底仓）/ `TECH_EXIT`（无先验→CLEAR）两分叉；`position_manager.refresh_state` 跨买入日按先验置 mode 正确。
- **依赖**：无（可与 T0.1 并行）；**功能价值依赖 T0.1**（卖出链通了先验才有意义）。

### T0.3 启动期 fail-closed：`account_id` / `mini_path`

- **目标**：消除「缺配静默回落假账户号 `UNKNOWN` / 空 userdata 路径」的真实资金口径污染风险。
- **涉及文件**：`app/run.py:219`（`account_id = ... or "UNKNOWN"`）、`:255`（`mini_path or ""`）、参照 `_build_calendar:203` 的 fail-closed 模式。
- **做什么**：在 `build_real_engine` 启动期断言 `settings.account_id` 非空、`settings.mini_path` 非空，缺配 `RuntimeError` 拒启动；移除 `"UNKNOWN"` / `""` 回落。保留「仅离线测试可显式放行」的口子（如有需要，仿日历的 `allow_*` 开关）。
- **验收标准**：单测覆盖「缺 `QMT_ACCOUNT_ID`/`QMT_MINI_PATH` → 启动期 RuntimeError」；配齐 → 正常装配。
- **依赖**：无。

### T0.4 健壮性加固（国金对接核对新增）

- **目标**：补齐核对发现的可离线加固项。
- **任务与文件**：
  1. **`_STATUS_NUM_MAP` 补 `255`**（`data_writer/normalize.py:105`）：`255: OrderStatus.REJECTED`（或保留兜底 REPORTED 但加注释说明 255=ORDER_UNKNOWN 故意留在途），避免 255 单长期挂在途致卖单 `SELLING` 不复位。
  2. **`cancel_order_stock` 加 try/except**（`order/order_executor.py:924`、`:973`）：包裹两处 cancel，异常留痕但不冒泡，避免打断 `sweep_expired` 整轮巡检（真机 cancel 对已成/不存在 order_id 的行为见 §A9，本加固无论结果都应做）。
  3. **SYNC 部分失败告警 + 重同步入口**（`storage/local_stack.py:123`、`app/scheduler.py:239`）：`sync_to_remote` 据 `report['ok']/errors` 分级日志（ok=False→`logger.error`/告警）；补「历史 `synced=0` 行手动重同步」运维入口（命令/接口）。
  4. **`LocalStorage` 父目录 makedirs**（`storage/local_stack.py`）：`db_path` dirname 非空时 `os.makedirs(dirname, exist_ok=True)`。
  5. **`order_remark` 长度复核**（`order/order_executor.py` `_REMARK_MAX_LEN`）：当前格式 ≈24 字符 > 官方建议 20；保留「截断也保住 `LUP|<日期>|` 头」逻辑，真机确认上限后（§A10）按券商实际值收紧。
- **验收标准**：每项补单测（255 状态映射、cancel 异常被吞不打断 sweep、SYNC ok=False 走 error 分支、makedirs 对不存在父目录可建表）。
- **依赖**：无。

### T0.5 死代码 / 惰性配置清理 + 注释纠偏（低优，可随手做）

- **目标**：消除误导性 API / 死代码 / 假可配置，降低后续维护误判。
- **任务与文件**：
  - `assert_safe_to_trade` 空 no-op 且无调用方（`config/settings.py:316`）：删除，或落地「竞价择时未实测不得开」的 fail-closed 校验并在启动期实际调用。
  - `PositionState.FROZEN` 无赋值点（`contracts/enums.py:124`）：补「单票浮亏击穿后持久冻结至人工恢复」赋值+恢复入口，或删除枚举与守卫。
  - `QMT_AUCTION_WINDOW`/`QMT_INTRADAY_SNAPSHOT_MINUTES`/`QMT_ACCOUNT_TYPE` 惰性配置：接线消费或从 `Settings` 删除（`QMT_ACCOUNT_TYPE` 结合 T1.1 的 `StockAccount` 实测再定，需要则透传，注意官方第二参是字符串）。
  - `try_next_best`/`next_best` 休眠死代码：维持现状（已文档化决策）或删除转次优分支。
  - **注释纠偏**：`connection_guard.py` / `xt_real.py:58` 的「QMT 无 on_connected」措辞改为「本引擎不依赖 on_connected，靠 connect() 返回值判就绪」（xtquant 实有 on_connected，设计无误仅措辞）；`protocols.py:132` `subscribe_quote` 默认 `'tick'` 加注释说明真实默认为 `'1d'`（调用处显式传参不影响行为）。
  - **MySQL 回落通道**（`app/run.py:122`）：二选一——实现 `conn_factory`（解析 DSN→`pymysql.connect`，仅 `qmt_*` 四表权限独立账号，敏感信息不入日志），或弃用（配 `QMT_MYSQL_DSN` 时启动期直接报错 + settings 注释标注不可用）。默认走 HTTP 不触发，**不阻塞上线**。
- **验收标准**：清理项不引入回归；保留项补注释说明「为何保留」。
- **依赖**：无。

## 2. 阶段 1 ｜ 真机实测固化（依赖国金 miniQMT 登录）

> 目标：上国金 Windows + miniQMT，用 `vars(obj)`/`dir(obj)`/实跑把占位/键名钉死，并完成卖出链的**完整盘口保真**。**全程 `KILL_SWITCH=true` 只观察、不下单（或最小单）**。

### T1.1 §A 占位常量 / 回报字段 / 行情结构确认（A1–A10）

- **目标**：把代码中「占位/待实测」逐项核对固化（核对结论：**预期均与代码占位一致**，本步是确认而非纠错）。
- **做什么**：按 [`待办 §A`](待办与上线验证清单.md) A1–A10 逐项：`get_full_tick` 键名/五档结构/volume 单位/竞价段 bidVol（A1–A4）；`xtconstant` STOCK_BUY/SELL/FIX_PRICE（A5）；回报对象字段名 + `order_status` 48–57（A6）；`order_stock` 参数顺序/`StockAccount` 签名/回调方法名（A7）；`order_stock` 失败返回 -1（A8）；`cancel_order_stock` 对已成/不存在 order_id 的行为（A9，决定 T0.4-2 是否够用）；`StockAccount` 是否需 `account_type` + `order_remark` 上限（A10）。
- **验收标准**：每项在目标机打印实际值核对，与代码一致即去掉「占位」注释钉死；不一致则改常量/键名 + 补单测。逐回调用「真实成交 / 主动撤单 / 必废单」验证 `on_stock_trade`/`on_cancel_error`/`on_order_error` 确被触发（方法名错无 getattr 兜底）。上线首笔最小买单核对 `qmt_trade.trade_side` 与实际方向一致。
- **依赖**：国金 miniQMT 客户端 + 账号就绪（见 [`待办 §G`](待办与上线验证清单.md)）。

### T1.2 跨帧盘口字段键名固化 + `SellBookBuilder` 完整保真

- **目标**：把 T0.1 保守置默认的 7 个跨帧字段真正算出来，让**破位/炸板/放量/连板了结**止损能力上线。
- **做什么**：用 A1 的真机 tick 核对「涨停价 / 分时均价（或自累计 VWAP） / 成交量量纲」键名；实现保留 per-code 帧历史的 `SellBookBuilder`（与 `AuctionPoller.prev_ticks` 同思路）派生 `is_sealed`/`broke_board`/`below_support`/`volume_surge`/`near_close_weak`/`open_times`/`price_volume_diverge`；替换 T0.1 的默认值。
- **验收标准**：先用真机只读盘口做 dry-run，逐字段核对派生正确（封板/炸板/破位场景各取真实样本对照）再放开；补单测覆盖跨帧状态机（封→炸板翻转计 open_times、封流比瓦解判 broke_board 等）。
- **依赖**：T0.1（最小接线）+ T1.1（A1 tick 键名）。

### T1.3 竞价择时能力实测（A4）+ 卖出链端到端

- **目标**：决定竞价择时能否开；端到端验证买入→持仓→卖出闭环。
- **做什么**：9:15–9:25 实跑观测虚拟封单 bidVol 是否有值（A4）→ 拿不到则竞价择时保持关、只用 last_price + 开盘后确认；端到端跑一遍（KILL_SWITCH 控制）验证买入成交→持仓回写→盘中卖出决策→下卖单→回流四表→看板闭环。
- **验收标准**：竞价择时开关据实测结论定稿；端到端无 ±8h 偏差、回流落库与看板一致。
- **依赖**：T1.1 + T1.2。

  T1.1 §A 占位/字段确认 ──▶ T1.2 跨帧字段保真（依赖 T0.1+T1.1）──▶ T1.3 竞价择时+端到端
